数据的两种形态

📊 结构化

让结构化数据易读易传

📋 规则系统（Rule-based）

1950-1980 年代 · 死记硬背的学生

核心思想

程序员手写所有规则，机器严格执行

实际例子：垃圾邮件过滤

规则代码

if 邮件包含("中奖"):
    标记为垃圾邮件
if 邮件包含("免费"):
    标记为垃圾邮件

✅ 优点：

  • 逻辑清晰
  • 结果可预测

❌ 缺点：

  • 规则写不完
  • 无法适应

🤖 机器学习（Machine Learning）

1990-2010 年代 · 会举一反三的学生

核心思想

给机器大量例子，让它自己总结规律

实际例子：房价预测

训练数据

面积地段房龄→价格
50㎡市中心3年→500万
100㎡市中心10年→800万
80㎡郊区5年→300万

学到：面积↑ 地段↑ → 价格↑

新房预测

输入：70㎡ + 市中心 + 2年

预测：约 550万

✅ 突破：机器自己学，不需写规则！

🧠 深度学习（Deep Learning）

2010+ · 能看懂图片、听懂语音

核心思想

模拟人脑神经网络，层层提取特征

实际例子：图片识别（识别猫）

🖼️ 照片

输出

✅ 突破：处理图片、声音等非结构化数据！

💬 大语言模型（LLM）

2020+ · 会表达思想的学生

核心思想

深度学习 + 海量文字 → 理解和生成人类语言

实际例子：智能问答

输入

"用一句话总结辛亥革命的意义"

↓ LLM 处理

输出

"推翻封建帝制，建立共和政体"

✅ 理解

语义

✅ 生成

文字

✅ 推理

思考

✅ 突破：能处理语言这种最复杂的非结构化数据！

LLM 也是一种系统！

同样遵循：输入 → 处理 → 输出

🧮 计算器系统

输入

125 + 75

→

处理

数学运算

→

输出

200

🤖 LLM 系统

输入

解释牛顿第一定律

→

处理

理解语义
生成回答

→

输出

保持静止或匀速运动

核心差异：处理的是"语言"而非"数字"

但本质上，LLM 仍然是一个系统！

• 有输入（文字）• 有处理（AI 模型）• 有输出（文字）• 可以通过 API 调用
